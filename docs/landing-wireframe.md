## 整体风格指引

### 色板（light mode，最终交付主色）

| 用途 | hex | 备注 |
| --- | --- | --- |
| 背景 base | `#FAF8F5` | 不是纯白，偏奶油的暖白（远离 Cursor 冷灰）|
| 背景 alt | `#F2EEE7` | section 分隔用 |
| 文本 primary | `#1B1A22` | 接近黑但偏紫，不是纯黑 |
| 文本 secondary | `#5C5867` | 副标 / sub copy |
| 文本 muted | `#8B8595` | hint / footer |
| accent purple | `#6B4FE0` | 主紫，CTA / 链接 hover / 强调字 |
| accent purple deep | `#4B33B8` | hover 态、icon stroke |
| accent purple soft | `#EAE3FF` | tag 底、卡片高亮底 |
| 边线 | `#E5DFD3` | 卡片 / 分隔线 |
| success | `#3A8A5C` | 「在做」「已对齐」标签 |
| warn | `#C7892F` | 数据可能不准 / 谨慎执行 |

**Pace 截图本身保留 dark theme**：截图边框 4px `#FAF8F5` 白底 + 12px `#EAE3FF` 紫色软光晕，把 dark UI 嵌在 light page 里反差感来源。

### 字体

| 角色 | 字体 | 字重 / size |
| --- | --- | --- |
| 显示标题（hero） | **Source Serif 4** | 600 / 64-72px |
| section 标题 | Source Serif 4 | 600 / 40-48px |
| 正文 | Source Serif 4 | 400 / 18px / line-height 1.65 |
| 强调字 / tagline | Source Serif 4 Italic | 400 italic |
| UI / 按钮 / sub | **Inter** | 500 / 14-16px |
| 命令 / 代码引用 | **JetBrains Mono** | 400 / 14px |

衬线给"沉静的 mentor 气质"，sans 给按钮的功能感，mono 只用在 `~/.claude/projects/...` 这种引用。

### 圆角 + 间距系统

- 圆角：卡片 `16px`、按钮 `12px`、tag `8px`、截图框 `20px`
- 间距单位：`8px` 基准（spacing: 8 / 16 / 24 / 32 / 48 / 64 / 96 / 128）
- section vertical padding：桌面 `128px`、移动 `64px`
- 内容最大宽：`1120px`，hero 副标最大宽 `640px`
- 卡片 padding：`32px`（feature 卡）/ `24px`（小卡）

### Dark mode

**首版 light only**。Pace app 本身是 dark，landing 是 light——刚好让截图站出来。dark mode toggle 留到 v2。

## 屏 3 — "Task tracker 救不了你"

```
+------------------------------------------------------------+
|                                                            |
|                                                            |
|         Task tracker 救不了你。                            |
|         你不缺一行待办，你缺一个先看完证据再说话的人。     |
|                                                            |
|                                                            |
|   Pace 不是：                                              |
|                                                            |
|   ╳ task tracker / Jira / Linear / 看板 / 甘特图           |
|   ╳ ChatGPT 聊天替代品                                     |
|   ╳ IDE 插件 / cc sidebar                                  |
|   ╳ multi-agent 编排内核                                   |
|   ╳ 给程序员的工具——它给所有重度用 cc 的职场人            |
|   ╳ 替你做决策的 boss——mentor 引导，不下命令              |
|                                                            |
|                                                            |
|   它做一件事：每次你开口，给你一段读完你工作的            |
|   mentor 回答——阶段判断 + 下一步 + 找谁怎么说。           |
|                                                            |
+------------------------------------------------------------+
```

**标题**: Task tracker 救不了你。
**Sub**: 你不缺一行待办，你缺一个先看完证据再说话的人。
**6 条反定义 copy**（直接对齐 PRODUCT.md §4，措辞收紧）:

1. **╳ task tracker / Jira / Linear / 看板 / 甘特图** — 没有 due date，没有燃尽图，没有 owner 列。
2. **╳ ChatGPT 聊天替代品** — 不闲聊，每次回答都基于你今天真实改过的代码和 transcript。
3. **╳ IDE 插件 / cc sidebar** — 独立 desktop dock，不抢 cc 上下文，不和 editor 抢焦点。
4. **╳ multi-agent 编排内核** — 不 spawn cc 子进程，不替你跑任务。它读你，不替你做。
5. **╳ 给程序员的工具** — 给所有重度用 cc 的人：PM / 设计 / 运营 / HR / 创业者。程序员只是其中一种。
6. **╳ 替你做决策的 boss** — mentor 引导，不下命令。它给阶段判断和选项，按哪个键是你的事。

**视觉建议**: 不用任何竞品 logo（版权 + 政治正确两头都省）。视觉用大号 `╳` 紫红色描边（不填充）+ 衬线粗体反定义文案。整屏背景换成 `#F2EEE7` alt 底，让这一屏在滚动节奏上有一个明确的"暂停"。

## 关键交互 + 动效建议

### Hero 动效

**不做 morph 动画 loop**（Granola 那种），原因：tagline 信息密度已经高，再加动效抢注意力。改用：

- Hero 截图：第一次进视口时 `opacity 0→1 + translateY 12px→0`（350ms ease-out），仅触发一次
- 4 个 hero 方案 review 期间：用一个 tab bar（A · B · C · D）切换，**不是** 4 个并排同屏，避免分散对比体验

### 屏 2 信号源流向

三个 chip → mentor → 输出气泡，**默认静态布局直接展示**，**不做 typewriter 动画**——把"按需读取 · 不后台常驻"这句话放下面就够诚实。如果一定要动效：进入视口时三个 chip 依次 `opacity 0→1`（每个 delay 100ms），结束就静止。

### 屏 4 carousel

- 默认每 6 秒自动切换（mobile 上）
- 桌面端默认静止，用户 hover 任意卡片暂停 + 高亮选中卡，配文跟随更新
- 左右箭头 + dots 控制；**不要** swipe-only（mobile 网页 swipe 体验差）
- 不做 ken burns（截图本身已经够紧凑，再缩放反而显廉价）

### Dark mode 切换

**首版不放 toggle**。整站 light，Pace 截图自带 dark，反差就是看点。等用户骂了再加。

### 滚动锚点

不做 sticky nav。header 只在 hero 区出现，滚出去就消失（典型 Linear 处理）。滚动顺畅、内容自己说话。
