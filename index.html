Task tracker 救不了你。

你不缺一行待办，你缺一个先看完证据再说话的人。

Pace 不是：

  • ╳

    task tracker / Jira / Linear / 看板 / 甘特图

    没有 due date，没有燃尽图，没有 owner 列。

  • ╳

    ChatGPT 聊天替代品

    不闲聊，每次回答都基于你今天真实改过的代码和 transcript。

  • ╳

    IDE 插件 / cc sidebar

    独立 desktop dock，不抢 cc 上下文，不和 editor 抢焦点。

  • ╳

    multi-agent 编排内核

    不 spawn cc 子进程，不替你跑任务。它读你，不替你做。

  • ╳

    给程序员的工具

    给所有重度用 cc 的人：PM / 设计 / 运营 / HR / 创业者。程序员只是其中一种。

  • ╳

    替你做决策的 boss

    mentor 引导，不下命令。它给阶段判断和选项，按哪个键是你的事。

它做一件事：每次你开口，给你一段读完你工作的 mentor 回答—— 阶段判断 + 下一步 + 找谁怎么说。

三个让人「噢，原来 mentor 该长这样」的瞬间。

不是 3 分钟视频，3 张静态图就够。

    见面前切到 ta 的视角，让 mentor 用 ta 的口气问你一遍——上桌前先排练完。

这是它的红线，也是它的承诺。

  • 本地优先

    你的 transcript 不进任何服务器，包括我们的。索引、缓存、配置全在 ~/.pace/，你随时能 rm -rf。

  • 你的 key

    自带 Anthropic / OpenAI / MiniMax key。Pace 不卖 token，不抽分成，不预留 SaaS 路径。

  • 被动应答

    不弹窗、不通知、不周一早上提醒你。你开口它才说话，关上窗就消失。

  • 桌面 dock

    独立 app（Electron），不是 IDE 插件，不抢 cc 焦点。锁屏快捷键召唤，写完答案缩回 dock。

  • MIT 开源

    代码全在 GitHub，issue 自己提，fork 自己改。坏话也可以骂在 PR 里。
